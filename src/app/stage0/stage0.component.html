Stage 0 - Landing Zone

What is a Landing zone?

Landing Zones allow you to standardize cloud environments for DevOps teams. They offer consistency across naming, scaling and access control along with a security baseline those accidental (or) non-compliant or unauthorized configurations. A Landing Zone should constantly evolve with core components of infrastructure and be well planned and maneuvered.

Consideration for Basic landing zone:

Principles and Guidelines to follow

The Cloud Adoption Framework enterprise-scale landing zone architecture represents the strategic design path and target technical state for an organization's Azure environment. It varies for organizations and are core recommendations and expected to be followed with variation based on the scenario.

Landing Zone Architecture for Enterprise Scale

Cloud Adoption Framework – Design Principle:

The design principles serve as a compass for subsequent design decisions across critical technical domains.

Subscription democratization

Subscriptions should be used as a unit of management and should be provided to business units to support the design, development, and testing of new workloads and migration of workloads.

Policy-driven governance

Azure Policy ensures continuous compliance and provides application owners with sufficient freedom and a secure unhindered path to the cloud.

Single control and management plane

Azure provides a unified and consistent control plane across all Azure resources and provisioning channels subject to role-based access and policy-driven controls.

Application-centric and archetype-neutral

Enterprise-scale architecture should focus on application-centric migrations and development rather than pure infrastructure lift-and-shift migrations.

Align Azure-native design and roadmaps

The enterprise-scale architecture approach advocates using Azure-native platform services and capabilities whenever possible.
