Stage 2 - Post Build QA

Stage Objectives

Check build readiness for testing and prepare the environment for subsequent test executions.

Commonly known as build smoke / sanity testing, verification testing, confidence testing (or) build acceptance testing, it has Developers and Testers focus on a set of tests adequate to ensure critical functionalities work as required in the build, confirming its stability and assurance to authorize if the QA team can proceed with further testing.

In this stage, you will explore the Post Build Quality Assurance stage of the overall testing pipeline for DevOps.

Pipeline Overview

Post-Build Quality Assurance flow:

This stage consists of two stages viz. Code Deployment and Smoke / Sanity Testing.

Deployment of Application

Leverage Azure Kubernetes Service (AKS) for deploying back-end and front-end container components of the application under test. This is achieved by pulling the Build Images from Azure Container Registry (ACR) and deploying it in AKS.

Smoke Testing

Balanced mix with minimal sub-set, but major features of API and UI based test automation in the form of Smoke / Build Verification Test is implemented using open source tools like Selenium and RestAssured with BDD Cucumber Framework. The automated test scripts is executed using Azure Pipeline and Reports published in Blob Storage.
