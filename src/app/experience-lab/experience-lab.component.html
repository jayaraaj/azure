Objectives & Premise

Objectives

Note: The lab focuses on only the execution stage of the testing lifecycle.

This section outlines the key objectives achieved by Practitioners on completion of the Experience Lab on Continuous Testing:

  • Choose various tooling alternatives, enablers and best practices in testing for DevOps
  • Identify types and stages of testing are essential for end-to-end quality assurance
  • Build and execute the continuous testing pipeline covering infra and pipeline as a code
  • Leverage the computing power of containers for seamless scalability in automation
  • Demonstrate a successful continuous testing implementation with Azure native services integrating with open source testing tools
  • Debug typical issues and fix your continuous testing pipeline

Application under Test

For this lab, we have considered an application prototype – Digital Shopify. It has a responsive web UI used for online shopping (Refer the Exhibit below). The current application is compatible with both desktop and mobile platforms and covers basic shopping workflows, providing feature such as:

  • Registration
  • Login
  • Product Search
  • Add to Cart
  • Enter Delivery Details
  • Checkout
  • Payment

Learnings

On completion of all lab stages, you will be able to -

  • Differentiate continuous testing from traditional testing
  • Propose key testing types for ensuring end-to-end quality assurance
  • Build and execute the continuous testing pipeline
  • Integrate open source automation tools in the pipeline
  • Help debug typical issues and fix the continuous testing pipeline
  • Leverage tooling alternatives, enablers and best practices in continuous testing

In order to get the best out of this workshop, we recommend you follow the sequence and build your pipeline in progression.
