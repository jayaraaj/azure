Stage 3 - Functional Assurance

Stage Objectives

Validate application’s functional behavior such as business process, user workflows & regression impact against the pre-defined outcomes.

Functional behaviours are “what” the system should do and Functional Testing evaluates if these functions perform as expected in the system. Functional requirements can be described and documented in business requirements specifications, epics, user stories, use cases, functional specifications (or) at times they may be undocumented. Functional tests should be performed at all test levels, though the focus is different at each level.

This stage will walk you through the assurance of your application functionalities. You will learn how to execute UI-based tests, responsive web design (RWD)-based tests and API tests using an open source tool stack.

The process consists of 2 steps: Build and Release.

Build

This will download source code from Azure Repository, create a Docker image of test framework and push it to Azure Container Registry. It will also produce a docker-compose.yml as a build artifact.

Release

The artifact published by build pipeline will trigger all the three tests in a parallel and publish test results.

With every code change committed to the repository, the Build Pipeline will be auto triggered and the subsequent Release pipeline will be triggered to execute the tests in parallel.

Automation for UI on Web and mobile with RWD tests is implemented using open source tools such as Selenium, Cucumber and Maven following Page Factory Pattern:

UI Automation:

UI test cases are executed automatically in the UI Test stage of the Release.

The corresponding manual test cases, which were created as part of landing zone, are also automatically updated.

RWD Automation

RWD test cases are executed automatically using Azure UI stage.

The corresponding manual test cases, which were created as part of landing zone, are also automatically updated.

API Tests

API test cases are executed automatically using Azure UI stage.

The corresponding manual test cases, which were created as part of landing zone, are also automatically updated.

Test Plan Integration

In the Landing Zone phase, Test Plans => Test Suites => Test cases are created. These test cases correspond to the Automated Smoke, UI, API and RWD tests.

Test Plan
