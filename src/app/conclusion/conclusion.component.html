    Jargons / CI Glossary

    • SDLC- Software development lifecycle is a process used to design, develop and test high quality softwares.
    • CT – Continuous Testing is the process of executing automated tests as part of the software delivery pipeline
    • CI/CD - Continuous Integration (CI) and Continuous Delivery (CD) bridges the gaps between development and operation activities and teams by enforcing automation in building, testing and deployment of applications.
    • QA – Quality Assurance process that ensures that an organization delivers the best products or services possible.
    • VMs- Virtual Machines is a virtual environment that functions as a virtual computer system with its own CPU, memory, network interface, and storage, created on a physical hardware system
    • AI- Artificial intelligence refers to the simulation of human intelligence in machines that are programmed to think like humans and mimic their actions.
    • DAST- Dynamic application security testing is a black box testing method that examines an application as it's running to find vulnerabilities that an attacker could exploit.
    • IAAS – Internet as a Service are cloud-based services, pay-as-you-go for services such as storage, networking, and virtualization
    • PAAS – Platform As a Service are hardware and software tools available over the internet
    • SAAS- Software as a Service are software that's available via a third-party over the internet.
    • UI – User Interface is anything a user may interact with to use a digital product or service.
